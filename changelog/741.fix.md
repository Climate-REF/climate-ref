The hash that identifies a diagnostic's input datasets is now computed deterministically across pandas versions and platforms. It previously relied on `pandas.util.hash_pandas_object`, whose output changes between pandas releases, which let a regression baseline minted in one environment fail the CI coupling gate in another and could make the solver re-run executions unnecessarily.

Because the underlying hash values change, the committed regression `catalog_hash` values have been regenerated, and existing databases will re-run each execution once on first use after upgrading.
