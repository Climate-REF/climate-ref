Fixed ``ref datasets ingest --n-jobs`` greater than 1 crashing the interpreter (SIGSEGV/SIGBUS) with the default ``complete`` CMIP6 parser. Parsing previously fanned out across threads, but the netCDF4/HDF5 wheels published on PyPI are not thread-safe, so opening files concurrently corrupted HDF5's global state. Parsing now runs in worker processes (using the ``spawn`` start method, matching the ``LocalExecutor``), which is safe and lets I/O-bound parsing scale across a parallel filesystem.
