ESMValTool reference data (the OBS, OBS6, native6 and obs4MIPs files in the provider's `data.txt` registry) is now ingested as the `esmvaltool-reference` source type, so it is recorded in the database. Selecting it at solve time is a separate follow-up. This also adds the CMIP5-era `OImon` sea-ice table to the MIP-table to frequency mapping, which the OSI-450 sea-ice references use.
